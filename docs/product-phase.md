# 成品化阶段路线

本文件用于定义项目从 MVP 进入成品化阶段后的默认推进顺序，并记录当前实际进度。

## 1. 当前阶段目标

在不破坏现有“项目管理 + 在线编辑 + 编译 + PDF 预览”主链路的前提下，引入更接近真实产品的能力：

1. 自动快照
2. 快照恢复
3. 成品化前端工作台
4. AI 问答助手、错误诊断与 inline completion
5. 协作编辑完整版
6. 存储层升级
7. 正式登录与工作空间平台化
8. 论文检索与 PDF 阅读
9. 探索页、模板入口与全局聚合搜索

## 2. 当前进度

### 2.1 已完成

- 核心编辑闭环：项目、文件、编译、PDF 预览
- 自动快照：成功编译快照、自动检查点、保护性恢复快照
- React 工作台：`React + TypeScript + Vite + CodeMirror 6`
- AI：单一 OpenAI 兼容 Provider、流式问答、结构化编译错误诊断
- AI 产品化：对话历史持久化、解释/优化快捷动作、修复代码生成、inline completion
- AI 上下文增强：`cased-kit`、`semchunk`、可选 `LLMLingua` 适配层
- 存储升级：`runtime-store` 统一元数据门面、`file/postgres` 可切换后端、迁移脚本
- 存储稳态增强：项目清单更新已切到原子 patch，文件化 JSON 写入已补齐原子落盘
- 正式认证：邮箱/密码注册登录、Cookie 会话、退出登录、鉴权中间层
- 工作空间：个人空间、组织空间、团队空间、团队项目直接可见
- 平台化存储：`file/s3` blob 门面、MinIO/S3 兼容参数、对象存储回填本地缓存
- 平台化编译：多 worker 原子领取、`host/docker` 双执行模式
- 平台观测：审计日志、版本事件查询与前端时间线面板
- 协作主键升级：`projectId + fileId`
- 评论与批注：批注、回复、解决状态、跳转定位
- 编译增强：`latexmk` 多轮编译与内容哈希缓存
- 协作编辑：`Yjs + WebSocket + CodeMirror 6`、在线状态、协作文档持久化
- 项目成员与邀请：本地演示用户会话、成员列表、邀请链接加入、项目访问过滤
- 编译体验增强：项目级主文件/引擎设置、任务配置快照、结构化诊断与日志跳转
- 论文模块：以“多源聚合检索 + 分层全文获取”为目标，当前已接入 `arXiv / PubMed / OpenAlex` 搜索，并打通项目文献库、`refs.bib` 导入、PDF 阅读与研究 Agent
- 论文服务拆分：主站已支持通过 `PAPER_ASSISTANT_BASE_URL` 对接独立论文搜索服务
- 轻回流：插入 `\cite{}`、插入论文总结、保存阅读笔记到项目文件
- PDF 摘录：高亮选区、项目级持久化、摘录列表回看与定位
- 探索页：模板筛选、模板详情预览、以模板创建项目

- 论文模板已接入基于官方类包的 `IEEEtran`、`acmart`、`elsarticle`、`llncs`
- 论文模板池已继续扩充到 `IEEE journal`、`ACM journal`、`Elsevier CAS`、`REVTeX 4.2`、`AASTeX 6.31` 与 `survey/supplementary/rebuttal`
- 官方模板镜像已新增 `CVPR / ICCV / 3DV` author kit，并在创建项目时落到本地缓存后再创建项目
- 官方模板镜像已继续扩到 `NeurIPS 2025`、`ICML 2025`、`ICLR 2026`、`AAAI 2025`
- 官方模板镜像已继续扩到 `ACL / EMNLP / NAACL`、`ECCV 2026`、`AISTATS 2025`、`COLM 2026`

## 3. 默认实施顺序

### 3.1 自动快照

第一版建议采用双触发策略：

- 成功编译后创建里程碑快照
- 编辑达到时间阈值后创建检查点快照

第一版建议：

- 先做全项目快照
- 先做整项目恢复
- 每项目保留最近 30 到 50 个快照

### 3.2 成品化前端工作台

当前已完成：

- 前端已切换为 `React + TypeScript + Vite + CodeMirror 6`
- 不再继续维护旧的原生 DOM 工作台
- 当前视觉基线继续参考 `考拉论文---在线-latex-协作平台/`

后续方向：

- 为协作编辑预留编辑器扩展点
- 继续收敛右侧结果区与编辑器操作流

### 3.3 AI 问答助手

当前已完成：

- 单一 OpenAI 兼容 Provider
- 流式问答
- 聊天接口已收敛为流式主入口
- 最近一次编译错误的结构化诊断
- AI 对话历史持久化
- 解释选中代码、优化选中代码、修复代码生成快捷动作

当前约束：

- 不实现多 Provider 平台
- 不在用户未确认时直接改写文件
- 上下文增强优先通过现有可选适配层实现，不单独起新的 AI 服务
- `LLMLingua` 这类重型压缩器默认保持可选，只有显式开启时才启用

### 3.4 协作编辑完整版

当前已完成：

- 同一项目文本文件的实时协作同步
- 在线状态与远端光标感知
- 文件切换时按 `projectId + filePath` 自动切换协作房间
- 协作文档状态持久化，服务重启后可恢复

后续方向：

- 评论锚点与 PDF 预览联动
- 更细粒度的权限与成员管理
- 协作事件审计与版本回放

### 3.5 项目成员与邀请系统

当前已完成：

- 项目显式成员关系、邀请链接创建/预览/接受/撤销
- 项目、编译、快照、AI、协作入口的成员访问控制
- 正式认证体系与持久登录
- 更细粒度项目角色：`owner/editor/commenter/viewer`
- 团队空间与组织级项目管理
### 3.6 存储层升级

当前文件化持久层已足够支撑开发，但成品阶段应逐步升级为：

- PostgreSQL 保存项目、文件、快照、会话等元数据
- 对象存储保存项目归档、快照包和编译产物

要求：

- 升级必须收敛在 `packages/runtime-store`
- 不允许让 API 和 Worker 感知底层存储细节

当前已完成：

- `runtime-store` 新增统一元数据存储门面
- 支持 `file` 与 `postgres` 元数据后端切换
- 项目、用户、邀请、AI 对话、编译任务、快照索引均已接入门面
- 项目 manifest 与通用 JSON 文件写入已增强为原子落盘，双烟雾测试并发下可稳定通过
- 已提供 `scripts/migrate-storage.js` 迁移脚本
- Blob 存储已支持 `file` 与 `s3(MinIO)` 两种后端
- `s3` 后端采用“本地缓存 + 对象存储同步”模式，避免影响现有项目树和编译工作目录

### 3.7 平台化能力

当前已完成：

- 正式登录、Cookie 会话与 `/api/me`
- 正式认证已完成本机 PostgreSQL 真实注册验证；当前开发机默认连接串为 `postgresql://overleaf:overleaf@127.0.0.1:5433/overleaf`
- 组织、团队、成员管理和工作空间列表接口
- 前端工作台可选择个人/组织/团队空间创建项目
- 审计日志与版本事件时间线面板
- 编译任务原子领取与多 worker 并发安全
- Worker 支持 `host/docker` 双执行模式
- Worker 已优先接入 `latexmk` 多轮编译
- 编译结果已支持基于项目内容哈希的缓存复用

### 3.9 论文检索与 PDF 阅读

当前已完成：

- 论文检索面板与项目文献库
- 论文模块产品方向已收敛为“多源聚合检索 + 分层全文获取”
- 当前已接入基于原生 `arxiv` 包的 `arXiv` 搜索
- 基于 LangChain `ArxivLoader` 的论文内容加载
- 当前已接入 `PubMed / OpenAlex` 搜索，并返回来源级搜索状态
- `OpenAlex` 阅读时会自动精确解析到 `arXiv / PubMed`，解析失败则保留 metadata-only 导入能力
- 研究场景 Agent/Tool，仅在论文检索与阅读界面使用
- 项目级 BibTeX 导入与 `refs.bib` 自动写入
- 论文 PDF 本地缓存与前端阅读器
- 论文 PDF 摘录与项目级持久化
- 论文阅读结果轻回流到写作区
- 论文阅读页右侧升级为 `Assistant + Notes` 双标签工作台
- 论文报告异步生成链路：`ensure/get/regenerate`、`paper-report worker`、全局报告缓存
- 结构化报告约束输出（章节完整性 + 证据锚点）与 `degraded` 降级语义

后续方向：

- 将独立论文搜索服务部署到香港等跨境访问更稳定的节点，主站通过 HTTP 调用
- 区分“搜索发现源”和“全文获取源”，避免把元数据源误当成全文源
- OpenAlex 显式标识补全与更强去重
- 从阅读摘录直接生成引用与写作片段

### 3.10 探索页与模板入口

当前已完成：

- 独立探索页入口，和写作工作台并列
- 平台精选模板与功能示例模板目录
- 模板详情预览、文件骨架预览与模板筛选
- 从模板直接创建项目并进入编辑器
- 顶栏全局聚合搜索，按来源分组展示结果

后续方向：

- 团队模板和私有模板接入
- 模板收藏、最近使用和模板版本管理
- 模板来源链接与可信度标识增强

### 3.8 评论与主键升级

当前已完成：

- 文件树、文件读取和协作房间已切到稳定 `fileId`
- 文件移动或重命名后，协作状态仍绑定原始文档而不是路径
- 评论与批注仓储已基于 `fileId` 建模
- 前端已支持添加批注、回复、解决状态和跳转定位

## 4. 当前阶段不优先事项

- 多 Provider AI 平台
- 多节点编译集群
- 复杂增量快照算法
- 高可用分布式存储

说明：

- 这里的“不优先”指的是“不直接跳到完整形态”，不是完全不做。
- 例如评论、权限和团队管理暂时不和协作内核一起一次性膨胀实现。

## 5. 决策原则

- 若一个方案能以更少抽象满足当前功能，应优先选择更简单方案。
- 若一个新能力会破坏现有编译和编辑主链路，应先修复边界，再引入新能力。
- 若一项能力只在远期才有价值，不应在当前阶段抢占实现顺序。
