# 项目自动快照模块 - 需求规格说明书

## 1. 项目概述

项目自动快照模块用于为在线 LaTeX 编辑器提供可靠的版本保留与恢复能力。该模块的目标不是实现完整的 Git 式版本控制，而是在当前成品化阶段为用户提供足够稳定、足够简单、足够可恢复的项目级快照能力。

该模块必须与现有的项目管理、编译链路和存储抽象协同工作，并遵循以下原则：

- 优先简单可靠，而不是复杂增量算法
- 优先项目级恢复，而不是一次性支持所有粒度
- 优先与现有编译和编辑主链路无缝集成

## 2. 设计原则

### 2.1 双触发，而不是单触发

自动快照第一版必须采用双触发机制：

1. 成功编译后创建里程碑快照
2. 项目在持续编辑后达到时间阈值时创建检查点快照

说明：

- 只在成功编译后快照会漏掉尚未编译或持续编译失败的关键编辑过程
- 只按时间周期快照会丢失“已验证可编译版本”这一高价值里程碑

### 2.2 先做全项目快照

第一版快照单位必须是整个项目目录，而不是单文件差异或块级增量。

说明：

- 整项目快照更容易验证一致性
- 整项目恢复路径更清晰
- 当前阶段复杂增量算法性价比过低

### 2.3 明确区分快照类型

第一版至少支持两类快照：

- `compile_success`：成功编译后的里程碑快照
- `auto_checkpoint`：编辑达到阈值后的检查点快照

## 3. 功能需求

### 3.1 快照创建

**用户故事：** 作为用户，我希望系统能自动保留关键版本，这样我在误操作或修改失控时可以回退。

#### 验收标准

- [ ] 当项目编译成功时，系统必须自动创建 `compile_success` 快照
- [ ] 当项目在持续编辑后达到时间阈值时，系统必须自动创建 `auto_checkpoint` 快照
- [ ] 当项目自上一次快照后没有内容变化时，系统不得创建重复快照
- [ ] 快照创建失败时，不得影响用户继续编辑和编译
- [ ] 快照元数据必须记录快照类型、项目 ID、触发来源、创建时间、内容摘要和存储位置

### 3.2 快照列表查询

**用户故事：** 作为用户，我希望查看项目的历史快照，这样我能了解哪些版本可以恢复。

#### 验收标准

- [ ] 用户可以查看指定项目的快照列表
- [ ] 快照列表必须至少显示创建时间、快照类型、摘要说明和来源
- [ ] 快照列表默认按创建时间倒序返回
- [ ] 用户可以区分快照是“编译成功版本”还是“自动检查点”

### 3.3 快照恢复

**用户故事：** 作为用户，我希望把项目恢复到某个历史快照，这样我可以撤销错误修改。

#### 验收标准

- [ ] 用户可以将整个项目恢复到指定快照
- [ ] 恢复操作前，系统必须先为当前状态创建一个保护性快照
- [ ] 恢复完成后，项目文件树和内容必须与目标快照一致
- [ ] 恢复操作不得破坏快照历史本身
- [ ] 恢复后用户可以立即继续编辑与编译

### 3.4 快照保留策略

**用户故事：** 作为系统维护者，我希望系统自动清理低价值旧快照，这样可以控制存储成本。

#### 验收标准

- [ ] 每个项目至少保留最近 30 个快照
- [ ] `compile_success` 快照保留优先级高于 `auto_checkpoint`
- [ ] 清理策略必须优先删除较旧的 `auto_checkpoint`
- [ ] 清理过程不得删除正在被恢复或正在创建的快照

## 4. 非功能需求

### 4.1 可靠性

- 快照创建失败不得影响项目保存
- 快照创建失败不得影响编译结果返回
- 恢复操作必须具备幂等性和明确错误反馈

### 4.2 性能

- 成功编译后触发快照时，不得显著阻塞编译结果返回
- 检查点快照的创建应尽量后台化
- 快照列表查询应在常规项目规模下快速返回

### 4.3 可维护性

- 快照逻辑必须集中在统一模块，不能分散在编译、编辑、前端多处
- 快照元数据与快照内容必须通过单一仓储层访问

## 5. 范围边界

### 5.1 当前阶段必须实现

- 自动创建快照
- 快照列表查询
- 整项目恢复
- 快照保留策略

### 5.2 当前阶段不做

- 块级或增量快照
- 单文件恢复
- 多人协作冲突下的细粒度版本合并
- 面向最终用户的复杂版本对比界面

## 6. 与现有模块的集成

### 6.1 与编译模块集成

- `workers/compiler` 在成功编译后触发里程碑快照创建

### 6.2 与编辑链路集成

- `apps/api` 或独立后台任务在检测到项目持续脏状态后触发检查点快照

### 6.3 与存储层集成

- `packages/runtime-store` 必须提供统一的快照仓储接口
- 快照元数据和快照内容的存取必须收敛在存储抽象层中
