# 项目成员与邀请模块需求

## 背景

当前项目已经具备文件级实时协作能力，但用户还缺少“如何进入同一个论文项目”的产品入口。因此需要补齐项目成员关系、邀请链接和访问控制。

## 目标

第一版目标：

1. 每个项目必须有一个所有者。
2. 项目支持 `owner` 和 `collaborator` 两种成员角色。
3. 所有者可以创建和撤销邀请链接。
4. 受邀用户可以通过邀请链接加入项目。
5. 项目列表只展示当前用户有权访问的项目。
6. 编译、快照、AI、协作连接都必须校验项目成员权限。

## 非目标

第一版不实现：

- 完整注册登录系统
- 邮箱邀请
- 组织/团队空间
- 复杂角色矩阵
- 审计日志与企业权限体系

## 验收标准

1. 新建项目后，创建者自动成为项目所有者。
2. 非成员无法读取项目、编译结果、快照和协作房间。
3. 所有者可在界面中看到成员列表和邀请链接列表。
4. 新用户通过邀请链接加入后，项目会出现在其项目列表中。
5. 成员与邀请数据在服务重启后仍然存在。
